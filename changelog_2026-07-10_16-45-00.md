## Bugs fixed (with verification)

### F1 — `analytics/pnl_service.py` — `timing_alpha` inverted for shorts 🔴
**Bug:** `timing_pnl = (nt_entry_price - entry_price) * qty` for both directions. For a SHORT a better entry is HIGHER than NT's, so the sign must flip; otherwise short timing alpha was reported inverted (Hermes penalised when it entered better).
**Fix:** `timing_pnl = entry_alpha if direction == "long" else -entry_alpha` (added `direction` param threaded from `record_realized_pnl`).
**Verified:** LONG Hermes@100/NT@105 → +5; SHORT Hermes@105/NT@100 → +5 (was -5). ✅

### F2 — `analytics/pnl_service.py` — `DrawdownTracker.get_stats()` duplicate equity ✅
**Bug:** `get_stats()` called `self.update(latest_equity, latest_ts)`, which APPENDED a duplicate entry to `_equity_history` on every read → corrupted equity curve / drawdown math.
**Fix:** Extracted `_compute_stats(equity, ts)`; `get_stats()` calls it WITHOUT mutating history.
**Verified:** history length stays 2 after `get_stats()`; max_dd_pct=0.05. ✅

### F3 — `monitor/cross_price.py` — NaN poisons correlation cache permanently ✅
**Bug:** `current_corr = np.corrcoef(...)` can be NaN (constant prices → zero variance). NaN was stored in `_last_correlations` permanently; all later delta comparisons against NaN are False → no shift ever fires for that pair.
**Fix:** `if not np.isfinite(current_corr): return None` before storing.
**Verified:** after 40 constant-price ticks, `_last_correlations` has no NaN keys. ✅

### F4 — `monitor/stop_watcher.py` — percentage trailing-stop gated on ATR (never fires) ✅
**Bug:** trailing update required `current_atr is not None`. The `percentage` method does NOT use ATR, so percentage trailing stops never updated.
**Fix:** only require ATR for `atr`/`brick_boundary` methods.
**Verified:** `_compute_trailing_stop(pos, 51000, None)` with `percentage` → 49980.0 (fires). ✅

### F5 — `signals/synthesizer.py` — re-feeds last 100 bars into renko (double volume) ✅
**Bug:** Every L4 cycle re-fed the same last 100 bars into the renko constructor → volume double-counted, bricks rebuilt from duplicate ticks.
**Fix:** Track `_renko_last_ts[sym]`; skip bars already fed.
**Verified:** logic + import. ✅

### F6 — `security/audit_logger.py` — stored XSS in HTML report ✅
**Bug:** Audit HTML report interpolated `username/resource/action/result` with no escaping → stored XSS when an operator opens the report.
**Fix:** `html.escape` on all interpolated fields.
**Verified:** `<script>` → `&lt;script&gt;` in rendered row. ✅

### F7 — `security/security_headers.py` — `HTTPSRedirectMiddleware` non-ASGI stub ✅
**Bug:** Class had `should_redirect`/`_is_https_request` but NO `__call__` → adding it as ASGI middleware did nothing.
**Fix:** Implemented real `async __call__(scope, receive, send)` that 301/308-redirects HTTP→HTTPS; added `set_app()`.
**Verified:** syntax + import. ✅

### F8 — `security/security_headers.py` — X-Forwarded-For trusted by default ✅
**Bug:** `HTTPSRedirectMiddleware(trust_proxy_headers=True)` default trusted proxy headers.
**Fix:** Default changed to `False`. (The actually-wired `security_headers_middleware` in `web/app.py` never reads XFF, so live risk was already minimal.)
**Verified:** import. ✅

### F9 — `docs/security_considerations.md`, `docs/secure_configuration_guide.md` — MFA/RBAC overstated ✅
**Bug:** Docs claimed MFA + RBAC as implemented features; code has only session-cookie auth (no MFA, no RBAC).
**Fix:** Docs corrected to state MFA / RBAC are NOT IMPLEMENTED (planned).
**Verified:** text replaced. ✅

### F11 — `web/app.py` — `/docs` `/redoc` `/openapi.json` exposed ✅

**Verified:** import + app constructs without docs. ✅
