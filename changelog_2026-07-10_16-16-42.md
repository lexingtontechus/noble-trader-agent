## Bugs fixed (with verification)

### F1 — `src/hermes/portfolio/state.py:261` — Equity double-counting 🔴 CRITICAL
**Bug:** `equity = cash + cash_usdc + gross_exposure + unrealized`. With the code's cash model (cash debited by full notional on entry, credited on close), `cash + gross_exposure = initial_cash + unrealized`. Adding `unrealized` again → `initial_cash + 2*unrealized` (double-counts PnL). Corrupted leverage, drawdown, peak-equity, margin.
**Fix:** `equity = cash_usd + cash_usdc + gross_exposure` (drop `unrealized`).
**Verified:** $100k acct, 1 BTC @$50k → $51k: equity = **101,000** (OLD BUG = 102,000). At entry = 100,000. ✅

### F2 — `src/hermes/portfolio/risk_gate.py:201-207` — Capping limits treated as blockers 🟠
**Bug:** A size-capped trade (approved_size > 0 after cap) was still rejected if any non-`volatility_cb_reduce` limit hit. Should approve at reduced size.
**Fix:** `approved = approved_size > 0` (a hard limit drives approved_size to 0 = true block; a cap reduces size but approves).
**Verified:** module imports; logic reviewed. ✅

### F3 — `src/hermes/execution/orchestrator.py:227 vs 351` — trade_id mismatch 🟠
**Bug:** Entry branch recorded `trade_id=order.trade_id`; exit branch recorded `trade_id=position.position_id`. The self-learning loop could not link entries↔exits.
**Fix:** Entry branch now uses `trade_id=position_id` (the id returned by `_register_position`), matching the exit branch.
**Verified:** module imports. ✅

### F4 — `src/hermes/portfolio/orchestrator.py:249` + `state.py` — Daily-loss CB fed cumulative PnL 🟠
**Bug:** `check_daily_loss(daily_loss_usd=metrics.realized_pnl)` used ALL-TIME cumulative PnL → circuit breaker effectively dead.
**Fix:** Added `realized_pnl_today` tracker to `PortfolioStateService` (resets at local midnight); orchestrator now passes `self._state.get_realized_pnl_today()`.
**Verified:** smoke test → `get_realized_pnl_today()` returns 1000 after a +$1000 close. ✅

### F5 — `src/hermes/agent/agent_ops.py:327` — SQL column `ts_closed` does not exist 🟠
**Bug:** `WHERE ts_closed >= ? AND ts_closed <= ?` — the `pnl_realized` column is `ts`. Query always errored → auto-rollback never fired (silent failure).
**Fix:** `WHERE ts >= ? AND ts <= ?`.
**Verified:** module imports. ✅

### F6 — `src/hermes/backtest/engine.py:399` — Monkeypatch restored in try, not finally 🟡
**Bug:** `migrate_mod.get_duckdb_path = original_get_path` was at the end of the try block; if an exception occurred mid-replay, the global stayed pointing at `temp_db` for the rest of the process.
**Fix:** Moved the restore into the `finally` block (alongside temp-DB cleanup).
**Verified:** module imports. ✅

### F7 — `src/hermes/backtest/engine.py` — equity per-heartbeat (consequence of F1) 🟡
Real via F1: `result.final_equity = metrics.equity_total` no longer double-counts, so per-heartbeat PnL in the optimizer is correct.

### F8 — `src/hermes/monitor/anomaly_detector.py:46` — wrong lookback comment 🟡
**Bug:** Comment claimed "1440 = ~60 days of 1m bars" (1440 min = 1 day, not 60).
**Fix:** Comment corrected to "1440 1m bars = 1 day of history". Value left as-is (1-day baseline is reasonable for 1m ticks); flag if 60-day intent was real.
