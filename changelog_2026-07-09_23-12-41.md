## 1. `src/hermes/transport/redis_subscriber.py` — Stream (XREAD) consumer

**Why:** The ingest subscriber used `pubsub.subscribe` (pub/sub). NT publishes qualified heartbeats to `signal.raw.noble_trader` as a **Redis Stream** (XADD). Pub/sub never sees stream entries → `ingest` received 0 records.

**What changed:**
- `_subscribe_loop()` rewritten to use `upstream.xgroup_create(name, group, id="0", mkstream=True)` then `upstream.xreadgroup(groupname, consumername="hermes-l0-worker", streams={channel: ">"}, count=10, block=5000)`.
  - `mkstream=True` creates the stream if absent (so the consumer group always exists).
  - `last_id=">"` reads only **new** messages in real-time (real-time tail). To replay backlog, set `id="0"`.
  - Each entry is `xack`'d after processing so restarts don't reprocess.
- New `_extract_payload(fields)` static method: NT may store the heartbeat as a single JSON field (`data`/`payload`/`heartbeat`/`message`) or as a flat key/value map. We prefer the JSON-bearing field; otherwise re-serialize the flat map to JSON so `parse_heartbeat` can validate it.
- Logging updated: `consumer_group_created`, `stream_read_started`, `stream_read_error`.

**Verified:** Test XADD of 3 schema-valid records → `ingest` logged `received=3`, `signal_heartbeats` accepted=3.

## 2. `src/hermes/backtest/engine.py` — price-driven replay + fixes

**Bug 1 — `regime_shift` type mismatch** (line ~245)
```python
# BEFORE
regime_shift=hb_data.get("regime_shift", "false"),
# AFTER
regime_shift=("true" if hb_data.get("regime_shift", False) in (True, "true") else "false"),
```
`NobleTraderHeartbeat.regime_shift` is a `Literal["true","false"]` string. The DuckDB column stores a BOOLEAN, so reads returned `True/False`; passing a bool raised Pydantic validation errors → all heartbeats rejected.

**Bug 2 — flat/synthetic pricing** (line ~279)
```python
# BEFORE — marked every position at the heartbeat entry_price only; no price path
for pos in portfolio_state.get_all_positions():
    portfolio_state.update_price(pos.symbol, hb.entry_price)

# AFTER — fetch REAL price series per symbol, walk candles, trigger SL/TP
from hermes.marketdata.price_feed import fetch_price_series
price_series = {sym: await fetch_price_series(sym, hours=72, config=self._config) for sym in symbols}
...
for _c in _ps.candles:
    portfolio_state.update_price(_pos.symbol, _c.close)
    if _sl and _c.low <= _sl: remove_position(..., "stop_loss_hit"); break
    if _tp and _c.high >= _tp: remove_position(..., "take_profit_hit"); break
```
Without this, every backtest returned identical `final_equity` (fee drag only) → the optimizer's objective was **constant** → 0 trials accepted.

**Bug 3 — no equity curve persisted** (in the candle walk)
Added per-candle `INSERT INTO account_snapshots (...)` into the **temp** backtest DB using `portfolio_state.get_metrics()`. This populates the tear sheet's equity curve so Sharpe/Sortino/drawdown are computed from real marks.
- **Fix:** timestamp column is `TIMESTAMPTZ`; passed `datetime.fromtimestamp(_c.ts_ms/1000, tz=timezone.utc)` (the raw `ts_ms` BIGINT caused `BIGINT -> TIMESTAMP` cast errors that silently dropped every snapshot).

**Bug 4 — backtest blocked by autonomy gate** (line ~189)
```python
risk_engine._autonomy_gate.set_paper_mode(True)  # added via autonomy_gate.py change below
```
Backtests are tier-0 per config; the live gate would otherwise require human approval and reject all simulated trades.

**Exit price fix** (line ~349)
Remaining positions now close at the symbol's last real candle close (`_ps.candles[-1].close`) instead of the heartbeat entry price.

## 3. `src/hermes/backtest/optimizer.py` — async + metrics + acceptance

**Bug 5 — `asyncio.run()` inside a running loop**
```python
# NEW helper
def _run_async(coro):
    try:
        asyncio.get_running_loop()
    except RuntimeError:
        return asyncio.run(coro)
    # already in a loop -> offload to a worker thread with its own loop
    import concurrent.futures
    def _worker():
        loop = asyncio.new_event_loop()
        try: return loop.run_until_complete(coro)
        finally: loop.close()
    with concurrent.futures.ThreadPoolExecutor(max_workers=1) as ex:
        return ex.submit(_worker).result()
# used in place of asyncio.run(...) in objective() and the grid-search fallback
```
`study.optimize()` runs inside `run_entry_timing_sweep`'s event loop; calling `asyncio.run()` there raised `RuntimeError: asyncio.run() cannot be called from a running event loop`.

**Bug 6 — metrics never populated**
```python
result.n_trades = t.get("n_trades", 0) or bt_result.n_fills   # real fills drive n_trades
```
The tear sheet's `trading.n_trades` reads `pnl_realized` (not written during replay). Falling back to `bt_result.n_fills` (which is correct: 3) makes `n_trades` real.

**Sharpe from equity curve** (fallback when `tear_sheet` returns 0)
Added a fallback that reads `account_snapshots.equity_total` from the backtest DB and computes an annualized Sharpe directly. This guarantees a non-flat objective.

**Acceptance gate** (small-sample handling)
- Full rigor checks (`run_rigor_checks` → deflated Sharpe, Monte Carlo) still require `n_trades >= 10`.
- For small samples (e.g. 3 heartbeats), `accepted = beat_baseline` and `rigor_checks_passed = None` (flagged N/A).
- **Note (verified behavior):** with only 3 heartbeats there is no entry-timing variation, so the sweep cannot beat the blind baseline → `Accepted: 0`. This is expected; acceptance requires more NT pushes over time.
